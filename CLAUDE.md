# CLAUDE.md

## 项目名称
科研文献溯源 Agent（Research Literature Trace Agent）

## 项目目标
构建一个基于 LangGraph 的参数化科研 Agent。用户输入一篇论文的 arXiv 链接后，Agent 自动完成文献调研、引用溯源、多轮迭代分析与最终阅读清单推荐。

## 核心任务
1. 读取用户输入论文。
2. 概括论文核心 idea。
3. 判定论文所属领域。
4. 结合 `related work` 与 `references`，筛选 top-k 篇值得深入调查的文献。
5. 对每篇候选文献递归执行相同分析。
6. 保留完整溯源轨迹。
7. 在迭代达到 m 轮或无可行方向后终止。
8. 调用 LLM 评估，选出最合理的一条文献脉络。
9. 输出“最值得阅读清单”与简要报告。

## 输入
- 一篇论文的 arXiv 链接

## 输出
最终输出为一个 Markdown 报告，至少包含：
- 最值得阅读清单
- 文献脉络说明
- 每篇论文的摘要概括
  - 重点描述：
    - 方法是什么
    - 试图解决什么问题

## 文献搜索与获取策略
- 默认优先通过 arXiv 获取论文。
- 若参考文献不在 arXiv：
  - 提示用户该文献无法直接通过默认来源获取。
  - 仍继续正常迭代搜索其他可行方向。
- 若某一分支没有更多可行方向，则该分支提前终止。

## 迭代与停止条件
以下参数应支持配置：
- `m`：最大迭代轮数
- `k`：每轮深入调查的 top-k 文献数量

停止条件：
- 达到最大迭代轮数 `m`
- 当前节点无法再找到可行候选文献
- 文献获取失败且无其他可探索方向

## 溯源轨迹设计
- 溯源轨迹采用 **graph 数据结构** 表示，而不是简单树结构。
- 同一篇文献在不同路径中出现时，应尽量复用节点与分析结果。
- 中间数据以 **JSON** 格式存储。
- graph 的具体实现尽量依赖成熟的开源 Python 库。

建议 graph 中至少包含：
- 论文节点信息
- 引用边信息
- 节点分析结果
- 节点状态（待分析 / 已分析 / 获取失败 / 跳过）
- 轮次信息
- 父子追踪关系

## 最终脉络评估标准
目标是从起源到 recent work 找到一条最合理的文献发展脉络。

优先评估维度：
1. **方法演进**
   - 关注“某工作采用某方法，是为了解决前作中的某问题”
   - 强调问题驱动的方法改进链条
2. **时间顺序**
   - 尽量符合领域发展的时间演进逻辑
3. **引用频次**（可选）
   - 如果难以稳定获取，可不作为强依赖

说明：
- 最终目标更偏向选出一条“发展链路”，而不是一个无边界的文献集合。
- 该链路应体现领域演进、方法更替和问题解决路径。

## LLM 设计要求
项目需支持多种 LLM 提供方，不能绑死单一厂商。

LLM 配置应抽象为统一接口，配置项至少包括：
- `base_url`
- `api_key`
- `protocol`
- `model_id`

要求：
- 可通过 config 切换不同模型/服务商
- 后续可扩展 OpenAI 兼容接口、Anthropic、以及本地模型服务

## 框架选型
- Agent 编排框架：**LangGraph**

## 设计原则
1. **参数化明确**
   - 关键流程、搜索深度、筛选数量、模型配置应可配置
2. **模块化**
   - 方便后续增加模块，如：
     - PDF 解析
     - 人工交互/人工确认
     - 更多文献源
     - 引用统计模块
3. **开源项目风格**
   - 清晰目录结构
   - 明确配置层与核心逻辑层
   - 良好的可测试性与可替换性
4. **可追踪**
   - 所有中间推理结果、筛选依据、节点关系应可回溯

## 建议的模块边界（草案）
- `ingestion`：输入论文获取与基础解析
- `summarization`：论文摘要与方法/问题抽取
- `retrieval`：参考文献提取与候选文献获取
- `ranking`：top-k 候选排序
- `graph_store`：图结构与 JSON 中间结果存储
- `tracing`：多轮溯源逻辑
- `evaluation`：最终文献脉络评估
- `reporting`：Markdown 报告生成
- `llm`：统一模型接口封装
- `config`：参数配置管理

## 当前明确的产品边界
本阶段先聚焦：
- 输入单篇 arXiv 论文
- 自动完成多轮引用溯源
- 输出一条最值得阅读的文献脉络及简要报告

暂不在第一阶段强依赖：
- 非 arXiv 全文统一抓取
- 高精度引用数统计
- 复杂人工交互工作流
- 前端界面

## 后续待细化问题
后续编码前仍可继续明确：
- graph 具体库选型
- 候选文献排序策略
- 论文信息抽取粒度
- LangGraph 中各节点与状态定义
- JSON schema 设计
- 报告模板格式
- 获取失败时的降级策略
